# Main Quest 1

### Main Quest 1. 바이브코딩으로 웹 서비스 만들기

<aside>
💡 바이브코딩을 활용해 직접 기획한 웹 서비스를 처음부터 끝까지 만들어봅시다! PRD를 작성하는 것부터 시작해서, 프론트엔드·백엔드·DB를 갖춘 하나의 웹 프로덕트를 완성하고 배포까지 해보는 것이 목표입니다. 지금까지 배운 프론트엔드, 백엔드, DB, 프레임워크, MCP, Skill 개념을 자유롭게 조합해서 '내가 만들고 싶었던 서비스'를 실제로 동작하게 만들어보세요.

</aside>

## 조건

### PRD 작성 (기획)

- 만들고 싶은 웹 서비스를 하나 정하고, 직접 PRD(Product Requirements Document)를 작성합니다.
- PRD에는 최소한 아래 내용이 포함되어야 합니다.
    - 문제 정의: 이 서비스가 어떤 문제를 해결하는가?
    - 타겟 유저: 누가 사용하는가?
    - 핵심 기능(MVP): 반드시 있어야 할 최소 기능 목록
    - 화면 구성: 어떤 페이지/화면이 필요한가?
    - 데이터 구조: 어떤 데이터를 저장하고 다룰 것인가?
- PRD를 먼저 작성한 뒤, 그 문서를 기준으로 바이브코딩을 진행합니다.

### 프론트엔드

- 사용자가 실제로 상호작용할 수 있는 화면을 구현합니다.
- 최소 2개 이상의 화면 또는 기능이 있어야 합니다. (예: 목록 화면 + 상세 화면, 입력 폼 + 결과 화면 등)
- 사용자의 입력이 화면에 반영되거나 서버 데이터를 불러와 보여주는 흐름이 있어야 합니다.

### 백엔드 / API

- 배운 프레임워크를 활용하여 API 서버를 구현합니다.
- 프론트엔드의 요청을 받아 처리하고 응답을 돌려주는 최소한의 CRUD(생성/조회/수정/삭제 중 일부)를 구현합니다.

### DB

- 데이터를 실제로 저장하고 다시 불러올 수 있도록 데이터베이스를 연동합니다.
- 서버는 별도로 제공하지 않습니다. 아래 중 편한 방법을 선택하세요.
    - 이미 사용할 수 있는 서버가 있는 경우: 본인의 서버/DB를 그대로 사용해도 됩니다.
    - 서버 준비가 어려운 경우: 무료 클라우드 서비스를 활용하세요. (DB 예시: Supabase, Firebase, Neon / 배포·서버 예시: Vercel, Netlify, Render, Railway, Cloudflare)

### 배포

- 로컬에서만 도는 것이 아니라, 실제로 접속 가능한 URL 형태로 배포합니다.
- 다른 사람이 링크만으로 서비스에 접근해 동작을 확인할 수 있어야 합니다.

### 진행 흐름 예시

- PRD 작성 → 화면/데이터 구조 설계 → 프론트엔드 구현 → 백엔드/API 구현 → DB 연동 → 배포 → 접속 확인
